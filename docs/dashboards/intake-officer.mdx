The dashboard uses tabs to organise <Term path="referral" plural /> by their current stage. Each tab displays a count badge showing the number of <Term path="referral" plural /> in that stage.

| Tab | Description | Badge |
|-----|-------------|-------|
| **Intake Processing** | New <Term path="referral" plural /> requiring initial processing | Shows count of <Term path="referral" plural /> in intake |

## Filters

Use the inline filters to narrow down your view:

| Filter | Options | Use Case |
|--------|---------|----------|
| **Search** | Free text search by client, referrer, or status | Find specific <Term path="referral" plural /> quickly |
| **Priority** | All Priorities, Emergency, Crisis, Urgent, Priority, Routine | Focus on high-priority <Term path="referral" plural /> |
| **<Term path="referral" /> Date** | All dates, Custom date range | View <Term path="referral" plural /> from specific time periods |

#### How to Use Filters

1. Type in the **search bar** to filter by client name, referrer, or status
2. Click the **Priority** dropdown to select a priority level
3. Click the **date** field to open the date picker and select a date range
4. The table updates automatically to show filtered results

## Intake Processing Tab

This is the default tab showing all <Term path="referral" plural /> that are new or in the initial intake processing stage.

### What You'll See

<Term path="referral" plural /> with the following statuses:
- **Draft** ‚Äî <Term path="referral" /> has been started but not yet submitted
- **Pending Consent** ‚Äî Awaiting client consent before proceeding
- **Pending Contact** ‚Äî Awaiting associated contact information
- **Pending Assessment** ‚Äî Awaiting assessment completion
- **Pending Cultural** ‚Äî Pending cultural and socioeconomic review

### How to Process New <Term path="referral" plural />

1. Review the list - check priority levels and age
2. Process Emergency and Crisis items first
3. Click the **eye icon** to view a <Term path="referral" />, or the **pencil icon** to edit
4. Complete the intake assessment
5. Submit for review when ready

<Callout color="teal" title="Reference Numbers">
Each <Term path="referral" /> has a unique reference number (e.g., REF-2026-DEFAULT-00047) that you can use to track and search for <Term path="referral" plural />.
</Callout>

---

## In Review Tab

This tab shows <Term path="referral" plural /> that have been submitted and are awaiting review by a supervisor or manager.

### What You'll See

<Term path="referral" plural /> with status:
- **Pending Review** - Submitted <Term path="referral" plural /> awaiting approval

A supervisor or manager must approve before the <Term path="referral" /> can proceed to service dispatch.

<Callout color="amber" title="Review Delays">
Use the **Age** column to monitor how long <Term path="referral" plural /> have been pending. If items stay in "Pending Review" for too long, follow up with your supervisor.
</Callout>

---

## Service Review Tab

This tab displays <Term path="referral" plural /> that have been approved and are ready to be allocated to services.

### What You'll See

<Term path="referral" plural /> with the following statuses:
- **Pending Service Dispatch** ‚Äî Approved <Term path="referral" plural /> ready to be dispatched to a service
- **Pending Service Review** ‚Äî <Term path="referral" plural /> awaiting service allocation review

### How to Dispatch a <Term path="referral" />

1. Click the **eye icon** on a <Term path="referral" /> to view details
2. Review the client's needs and service requirements
3. Select the appropriate service
4. Assign to a <Term path="caseWorker" /> or team
5. Confirm the dispatch

---

## Awaiting Service Delivery Tab

This tab shows <Term path="referral" plural /> that have been dispatched to services and are in various stages of service delivery.

### What You'll See

<Term path="referral" plural /> with the following statuses:
- **Pending Allocation** ‚Äî Accepted by a service, awaiting worker allocation
- **Awaiting Service** ‚Äî Allocated to a worker, service has not started yet
- **Active** ‚Äî Service delivery is in progress
- **Completed** ‚Äî Service delivery has been completed
- **Cancelled** ‚Äî <Term path="referral" /> was cancelled
- **On Hold** ‚Äî <Term path="referral" /> is temporarily paused
- **Closed** ‚Äî <Term path="referral" /> has been permanently closed

### Status Filter Dropdown

This tab includes an additional **status filter dropdown** that lets you narrow the list to a specific post-dispatch status (any of the statuses listed above, or "All Statuses").

### Monitoring Post-Dispatch <Term path="referral" plural />

- Track <Term path="referral" plural /> that are awaiting service start using the Age column
- Use the status filter dropdown to focus on a specific stage
- Follow up on delayed service initiations
- Review completed <Term path="referral" plural /> for closure

Clicking any analytics card opens an expanded panel below it. Click the **X** button to close.

- **Active Pipeline** ‚Äî Shows all <Term path="referral" plural /> across every tab.
- **New <Term path="referral" plural />** ‚Äî Combines <Term path="referral" plural /> from the Intake Processing, In Review, and Service Review tabs into one list.
- **Needs Attention** ‚Äî Shows only Intake Processing and In Review <Term path="referral" plural />, filtered to Urgent/Crisis/Emergency priority and sorted with Emergency first.

## Troubleshooting

### Dashboard Not Loading

1. Check your internet connection
2. Click the refresh button
3. Try logging out and back in
4. Clear your browser cache

### Missing <Term path="referral" plural />

If you can't find a <Term path="referral" />:
1. Check all tabs (it may have moved to a different status)
2. Use the search function with the reference number
3. Check the filters - you may have a filter active
4. Go to the full <Term path="referral" plural /> page for advanced search

### Data Not Updating

1. Click the refresh button
2. Check if filters are affecting the view
3. Wait a few moments and try again
4. Contact support if the issue persists
