# Intake Officer Role

Understand the intake officer role, including capabilities, responsibilities, and dashboard features.

## Overview

Intake Officers are responsible for managing the referral intake process - from receiving new referrals to dispatching them for service delivery. They serve as the gateway for clients entering the Manaaki Central system.

## Key Responsibilities

### Referral Processing
- Receiving and reviewing new referrals
- Verifying client information accuracy
- Assessing referral priority and urgency
- Ensuring all required documentation is collected

### Assessment & Review
- Conducting initial intake assessments
- Reviewing consent documentation
- Evaluating service eligibility
- Identifying appropriate services for clients

### Service Dispatch
- Matching clients with appropriate services
- Coordinating with service teams
- Managing referral workflow progression
- Ensuring timely service allocation

### Documentation
- Maintaining accurate referral records
- Uploading and organizing supporting documents
- Recording notes and observations
- Tracking referral history

## Permissions

| Action | Permission |
|--------|------------|
| View all referrals | Yes |
| Create new referrals | Yes |
| Edit referral details | Yes |
| Submit referrals for review | Yes |
| Approve/reject referrals | Yes (within scope) |
| Dispatch to services | Yes |
| View client information | Yes |
| Upload documents | Yes |
| Delete referrals | No (soft delete only) |
| Access other dashboards | Limited |

## Dashboard Features

The Intake Officer Dashboard provides a comprehensive view of the referral pipeline:

- **Analytics Overview** - Key metrics showing referral volumes and statuses
- **New Referrals** - Referrals in draft or initial processing stage
- **In Review** - Referrals pending supervisor or manager review
- **Service Review** - Referrals awaiting service allocation
- **Post-Dispatch** - Referrals that have been allocated to services

For detailed dashboard documentation, see [Intake Officer Dashboard](/dashboards/intake-officer).

## Workflow Overview

```
New Referral → Draft → Submitted → In Review → Service Review → Dispatched → Active/Completed
```

### Status Progression

1. **Draft** - Referral created but not yet submitted
2. **Submitted** - Referral submitted for processing
3. **Pending Review** - Awaiting supervisor review
4. **Pending Service Review** - Approved, awaiting service allocation
5. **Awaiting Service** - Dispatched, waiting for service to begin
6. **Completed** - Service delivery completed

## Best Practices

### Prioritization
- Process **Urgent** and **Priority** referrals first
- Review the priority queue at the start of each day
- Escalate urgent cases that require immediate attention

### Data Quality
- Verify client contact information
- Ensure all required fields are completed
- Check for duplicate client records before creating new ones

### Communication
- Document all client communications
- Keep referrers informed of progress
- Coordinate with supervisors on complex cases

### Timeliness
- Aim to process new referrals within 24-48 hours
- Follow up on stalled referrals
- Monitor the In Review queue for delays
